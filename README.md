# Система учета заявок на ремонт климатического оборудования

Веб-приложение для автоматизации процесса обработки заявок на ремонт климатического оборудования. Разработано для ООО "Конди" сервисным центром "IT-Сфера".

## Быстрый старт

```bash
# Установка зависимостей
npm install

# Инициализация базы данных
node database/init_database.js

# Запуск приложения
node server.js
```

После запуска откройте браузер и перейдите по адресу: **http://localhost:3001**

## Тестовые аккаунты

Для входа в систему используйте следующие учетные записи:

| Роль | Логин | Пароль | Описание |
|------|-------|--------|----------|
| Менеджер | `login1` | `pass1` | Полный доступ к системе |
| Специалист | `login2` | `pass2` | Выполнение ремонтных работ |
| Оператор | `login4` | `pass4` | Обработка заявок |
| Заказчик | `login7` | `pass7` | Создание заявок |

## Основные возможности

**Управление заявками**
- Создание заявок с указанием типа и модели оборудования
- Автоматическое присвоение номеров заявок
- Отслеживание статуса выполнения работ
- Назначение ответственных специалистов

**Система ролей**
- 5 ролей пользователей с разными правами доступа
- Контроль доступа к функциям системы
- Персональные дашборды для каждой роли

**Отчеты и аналитика**
- Статистика по заявкам и типам оборудования
- Среднее время выполнения работ
- Анализ эффективности специалистов

**Оценка качества**
- Генерация QR-кодов для завершенных заявок
- Форма обратной связи с тремя вариантами оценки
- Сбор комментариев от клиентов

## Роли пользователей

**Заказчик**
- Создание заявок на ремонт
- Просмотр статуса своих заявок
- Добавление комментариев

**Оператор**
- Обработка всех заявок в системе
- Назначение специалистов на заявки
- Изменение статусов заявок

**Специалист**
- Просмотр назначенных работ
- Обновление статуса выполнения
- Добавление комментариев о ходе работ

**Менеджер**
- Полный контроль над системой
- Доступ к отчетам и статистике
- Управление процессом обработки заявок

**Менеджер по качеству**
- Контроль качества выполнения работ
- Консультации при сложных случаях
- Продление сроков выполнения заявок

## Жизненный цикл заявки

1. **Новая заявка** - создана заказчиком, ожидает обработки
2. **В процессе ремонта** - назначен специалист, работа началась
3. **Ожидание комплектующих** - работа приостановлена в ожидании запчастей
4. **Готова к выдаче** - ремонт завершен, можно забирать оборудование
5. **Завершена** - оборудование выдано клиенту, заявка закрыта
6. **Отменена** - заявка отменена по запросу клиента

## Система оценки качества

После завершения работ система автоматически генерирует QR-код, который ведет на форму оценки качества. Клиенты могут выбрать один из трех вариантов:

- **Хорошо** - работа выполнена качественно, в срок, без замечаний
- **Нормально** - работа выполнена удовлетворительно, есть небольшие замечания
- **Плохо** - работа выполнена некачественно, есть серьезные замечания

Дополнительно можно оставить текстовый комментарий с подробным описанием.
